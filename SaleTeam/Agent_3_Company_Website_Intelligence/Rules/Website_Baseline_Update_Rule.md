# Quy tắc: Khi nào cập nhật baseline website

## Mục tiêu
Giúp Agent 3 quyết định khi nào cần cập nhật lại baseline thay vì chỉ ghi thêm change log.

## Nên cập nhật baseline khi
- headline chính đổi rõ rệt
- CTA chính đổi rõ rệt
- product / solution structure đổi đáng kể
- pricing page xuất hiện, biến mất hoặc thay đổi logic định giá
- website đổi định vị thị trường, ngành dọc hoặc ICP mục tiêu
- website thêm nhiều case study, customer logo, partner page hoặc newsroom mới làm thay đổi cách đọc account
- website đổi cấu trúc sitemap ở mức đáng kể

## Không cần cập nhật baseline khi
- chỉ sửa câu chữ nhỏ
- thêm 1 bài blog thông thường không đổi định vị
- thay đổi nhỏ ở footer, nav hoặc meta content
- cập nhật tin tức lẻ không làm đổi cách hiểu về account

## Quy tắc thực thi
- Nếu chưa có baseline: tạo baseline, không ghi đây là `thay đổi`.
- Nếu có thay đổi lớn: ghi change log trước, sau đó cập nhật baseline.
- Nếu thay đổi nhỏ: chỉ ghi change log nếu vẫn có giá trị cho sales; không bắt buộc cập nhật baseline.
- Nếu không chắc thay đổi có đủ lớn không, giữ baseline cũ và gắn nhãn `Cần rà lại`.
